options:
  -h, --help            show this help message and exit
  --source SOURCE       Video input source, defaults to webcam
  --save [PATH]         Save annotated video. Omit a value to use
                        Outputs/Video/[stem]_Video_Output.mp4, or supply a custom path.
  --log LOG
  --summary [PATH]      Save post-run summary CSV. Omit a value to use Outputs/CSV
                        Files/[stem]_summary.csv, or supply a custom path.
  --heatmap [PATH]      Save per-participant scene gaze heatmaps. Omit a value to use
                        Outputs/heatmaps/[stem]_Heatmap_Output (one PNG per participant), or
                        supply a custom directory/prefix path.
  --charts [PATH]       Generate post-run time-series charts for each phenomena tracker. Omit a
                        value to use Outputs/Charts/[stem]_Charts.png, or supply a custom path.
  --save-detections     Write a per-frame detections side stream ([stem]_detections.csv next to
                        the summary CSV: one row per detection with class, confidence, and bbox).
                        Off by default; the validation suite enables it for its runs to score
                        object IoU.
  --pipeline YAML       Load pipeline configuration from a YAML file. CLI flags override YAML
                        values.
  --project DIR         Run in project mode: process all staged videos (DIR/Inputs/Runs/ run
                        folders, or the legacy flat DIR/Inputs/Videos/) using
                        DIR/Pipeline/pipeline.yaml as config.
  --no-resume           Project mode: reprocess every video, ignoring the resume ledger. Does not
                        archive prior outputs. Default resumes -- skipping videos whose ledger
                        entry is done with an unchanged config.
  --preflight           Print the project readiness report (requires --project) and exit. Checks
                        pipeline config, weights, VP file, runs, participants/conditions, device,
                        disk, and plugin load errors. Exit 0 if no failures, else 1.
  --participant-ids IDS
                        Comma-separated participant labels for single-video mode. Positional:
                        first label maps to track 0, second to track 1, etc. E.g. --participant-
                        ids S70,S71,S72
  --participant-csv CSV
                        Path to a participant_ids.csv mapping video filenames to custom
                        participant labels (see docs for format).
  --aux-stream SOURCE:VIDEO_TYPE:LABEL:PIDS
                        Auxiliary video stream. Format: SOURCE:VIDEO_TYPE:LABEL:PID1,PID2 where
                        SOURCE is the file path, VIDEO_TYPE is one of
                        eye_only/face_closeup/wide_closeup/custom, LABEL is a user-defined stream
                        label, and PIDS is a comma-separated list of participant labels.
                        Repeatable for multiple streams.
  --aux-auto-detect     Enable automatic face detection on wide/face auxiliary streams (default:
                        enabled).
  --device {auto,cpu,cuda,mps}
                        Compute device for all backends: auto, cpu, cuda, or mps. 'auto' selects
                        CUDA > MPS > CPU (default: auto).
  --anonymize [{blur,black}]
                        Anonymize faces in the output video: 'blur' applies heavy Gaussian blur,
                        'black' fills with a solid rectangle. Bare --anonymize means 'blur'.
  --overlay-theme {classic,mindsight}
                        Annotated-frame overlay styling: 'classic' is the historical high-
                        saturation look, 'mindsight' uses the brand palette (indigo ink label
                        tabs, logo magenta/jade accents). Cosmetic only -- analysis outputs are
                        unaffected (default: classic).
  --anonymize-padding FRAC
                        Fraction of face bbox size added as padding for anonymization (default:
                        0.3).
  --model MODEL         Object Detection Model (default: yolo11n.pt since v1.3; yolo11n.onnx in
                        the Models tab is a faster option for CPU-bound installs)
  --face-conf T         RetinaFace face-detection confidence threshold (default: 0.5, the library
                        default). Lower to catch small/distant faces.
  --face-input-size PX  RetinaFace square input resolution in pixels (default: 640, the library
                        default). Raise for small/distant faces at a speed cost.
  --face-model NAME     RetinaFace backbone variant (default: r34 -- the v1.3 eval pick: best
                        accuracy AND ~40% faster than mnet_v2 on the CoreML path; weights auto-
                        download on first use).
  --conf CONF           Object detection confidence threshold, defaults to 0.35
  --classes CLASSES [CLASSES ...]
                        Specified YOLO Object Detection Classes Prompt
  --blacklist BLACKLIST [BLACKLIST ...]
                        Specified YOLO Object Detection Classes Blacklist
  --skip-frames SKIP_FRAMES
                        Frames between object detection. Higher = faster but less accurate.
                        (Defaults to 1, i.e. process every frame)
  --detect-scale DETECT_SCALE
                        Detection scale for Object Recognition
  --vp-file VP_FILE     Path to visual prompt file for use with YOLOE object detection models
  --vp-model VP_MODEL   YOLOE model to use alongside visual prompting for object detection

                        Use the FULL visual prompt everywhere, ignoring condition tags in the VP

  --no-detector         Run without any object-detection model: faces, gaze rays, and gaze-tip

  --obj-persistence N   Dead-reckon object bboxes for N frames after they disappear (default 0).

  --merge-overlap-strategy {filter,merge,dynamic}

  --lite-overlay        Minimal overlay: disable cone rendering, convergence markers, dwell arcs,

  --no-dashboard        Skip dashboard composition for maximum throughput. Displays the raw

Ray Forming (Gazelle blend):
  --rf-gazelle-model PATH
                        Path to a gaze-target model for Gazelle blend ray forming: a Gaze-LLE
                        checkpoint (.pt, torch) or a gazelle-dinov3 ONNX export (.onnx,
                        onnxruntime). Used alongside a pitch/yaw backend (MGaze, etc.) to
                        periodically correct rays with scene-aware heatmaps.
  --rf-gazelle-name NAME
                        Gaze-LLE model variant for ray forming (default: gazelle_dinov2_vitb14).
  --rf-inout-gate T     Activate the Gaze-LLE checkpoint's in/out-of-frame head (when present) and
                        gate the blend with it: heatmap accepts with an in-frame score below T are
                        vetoed and blend trust is scaled by the score. 0 disables (default: 0.0).
  --rf-reuse-eps EPS    Skip a scheduled Gaze-LLE call when the scene is visually unchanged since
                        the last real call (mean-abs 64x64 grayscale frame diff below EPS and
                        stable face boxes); the cached heatmaps are re-anchored instead. 0
                        disables (default: 0.0). Saves forward passes on static footage.
  --rf-onset-samples N  Let a NEW face reach fixation eligibility after N pitch/yaw samples
                        instead of the full warmup of 5 (half the history buffer). Minimum
                        effective value 2; 0 keeps the full warmup (default: 3). Cuts first-anchor
                        latency for freshly appeared faces.
  --rf-onset-gap N      When a never-anchored face wants inference, relax the global call gap to
                        min(--min-call-gap, N) so a new participant is not stuck behind another
                        face's recent call. 0 disables (default: 5).
  --rf-len-refresh-gap N
                        Every N frames, run a cheap half-precision Gaze-LLE pass that refreshes
                        ray LENGTH only -- direction stays with the full-precision fixation-gated
                        corrections. Keeps ray reach fresh between corrections at a fraction of
                        the cost. 0 disables (default: 10).
  --rf-len-slew N       When a length refresh re-latches ray length, ramp the effective length
                        target from the reach currently shown to the new value over N frames
                        instead of snapping, with the length-hold decay paused during the ramp
                        (monotone approach, no bounce). 0 = instant snap (default: 5).
  --rf-len-gain F       Scale the blend ray-length target by F. The v1.3 eval decomposition
                        measured a systematic under-reach (84% of rays short); 1.10 recovered ~5
                        px mean / ~8 px median on the eval labels. Applied before smoothing so
                        snap and hit detection see the corrected reach. 1.0 = off (default: 1.10).
  --rf-endpoint-extract {centroid,topp}
                        How the Gaze-LLE heatmap becomes a ray-length endpoint: 'centroid' = full-
                        map mass centroid (historical), 'topp' = mass centroid of the top-50%-mass
                        cells -- diffuse or multi-modal heatmaps no longer drag the endpoint
                        toward the origin (default: centroid).
  --rf-gazelle-fp16     Run the blend-path Gaze-LLE model in half precision (CUDA/MPS only;
                        ignored on CPU). Faster per call; output is NOT byte-identical to fp32, so
                        keep it off for regression baselines.
  --rf-gazelle-compile  torch.compile the blend-path Gaze-LLE model (PyTorch 2+, experimental --
                        MPS support immature). First call pays a warmup cost.
  --rf-gazelle-interval N
                        (Legacy) alias for --min-call-gap. If both are given, --min-call-gap wins.
                        Kept so old scripts/YAMLs keep working (default: unset -> min_call_gap
                        default).
  --min-call-gap N      Minimum frames between Gaze-LLE inference calls. The scheduler also
                        requires at least one participant to be fixating before firing (default:
                        30).
  --fixation-v-threshold F
                        Smoothed pitch/yaw velocity (rad/frame) at which a face is treated as 50%
                        fixating. Lower = safer anchoring (default: 0.04).
  --fixation-d-threshold F
                        Windowed pitch/yaw dispersion (rad) at which a face is treated as 50%
                        fixating (default: 0.15).
  --dir-min-cutoff F    Direction One Euro smoother floor cutoff (Hz). Lower = smoother at rest
                        (default: 1.0).
  --dir-beta F          Direction One Euro responsiveness. Higher = snaps to fast motion (default:
                        0.5).
  --len-min-cutoff F    Length One Euro smoother floor cutoff (Hz) (default: 1.0).
  --len-beta F          Length One Euro responsiveness. Lower than direction by default so length
                        holds steadier (default: 0.3).
  --len-hold-tau F      Seconds the Gaze-LLE-derived ray length persists after an accepted
                        inference before decaying back to the pitch/yaw baseline. Direction
                        reverts quickly on its own; raise this to hold ray reach longer between
                        inferences (default: 5.0).
  --depth-ray-length    Use depth map to scale ray length based on scene geometry (default: off).
  --depth-length-min F  Ray length multiplier at depth=0 (nearest) (default: 0.5).
  --depth-length-max F  Ray length multiplier at depth=1 (farthest) (default: 3.0).
  --depth-belief-boost F
                        How much depth agreement boosts Gaze-LLE heatmap confidence in the belief
                        update (default: 0.0).

  --gazelle-model PATH  Path to a Gazelle checkpoint (.pt). Activates the Gazelle backend.
  --gazelle-name NAME   Gazelle model variant (choices: gazelle_dinov2_vitb14,
                        gazelle_dinov2_vitb14_inout, gazelle_dinov2_vitl14,
                        gazelle_dinov2_vitl14_inout; default: gazelle_dinov2_vitb14).
  --gazelle-inout-threshold F
                        In/out-of-view confidence threshold for *_inout model variants. No effect
                        on non-inout models (default: 0.5).
  --gazelle-device {auto,cpu,cuda,mps}
                        Compute device: auto, cpu, cuda, or mps. 'auto' selects CUDA > MPS > CPU
                        (default: auto).
  --gazelle-skip-frames N
                        Reuse the previous gaze result for N frames between inference runs. 0 = no
                        skipping (default: 0).
  --gazelle-fp16        Use half-precision (float16) inference on CUDA/MPS (ignored on CPU).
  --gazelle-compile     Use torch.compile() for the Gazelle model (PyTorch 2.0+ only).

  --mgaze-model MGAZE_MODEL
                        Path to MGaze model weights (.onnx or .pt)
  --mgaze-arch {resnet18,resnet34,resnet50,mobilenetv2,mobileone_s0,mobileone_s1,mobileone_s2,mobileone_s3,mobileone_s4}
                        Architecture name (required for .pt models)
  --mgaze-dataset MGAZE_DATASET
                        Dataset config key (default: gaze360)

Gaze Boost plugin:
  --gaze-boost          Boost confidence of detections near gaze endpoints.
  --gaze-boost-factor GAZE_BOOST_FACTOR
                        Multiplicative confidence boost (default: 1.5).
  --gaze-boost-radius GAZE_BOOST_RADIUS
                        Pixel radius around gaze endpoints (default: 100).
  --gaze-boost-min-conf GAZE_BOOST_MIN_CONF
                        Minimum YOLO conf for sub-threshold candidates (default: 0.10).
  --gaze-boost-max-conf GAZE_BOOST_MAX_CONF
                        Cap on boosted confidence (default: 0.95).
  --gaze-boost-classes GAZE_BOOST_CLASSES [GAZE_BOOST_CLASSES ...]
                        Only boost these class names (default: all non-person classes).

Eye Movement plugin:
  --eye-movement        Enable eye movement classification.
  --em-source {gaze,iris}
                        Velocity source (default: gaze).
  --em-saccade-thresh F
                        Saccade threshold px/frame (default: 30.0).
  --em-fixation-thresh F
                        Fixation threshold px/frame (default: 10.0).
  --em-min-fixation N   Min fixation duration frames (default: 4).
  --em-velocity-window N
                        Median filter window (default: 3).

Novel Salience plugin:
  --novel-salience      Enable novel-salience detection (rapid gaze-shift tracking).
  --ns-speed-thresh PX  Gaze-endpoint speed threshold in pixels/frame to flag a novel-salience
                        event (default: 40). Lower = more sensitive.
  --ns-cooldown N       Minimum frames between consecutive novel-salience events for the same face
                        track (default: 20).
  --ns-history N        Sliding-window depth for velocity smoothing (default: 2). 1 =
                        instantaneous; 3 = heavier smoothing.
  --ns-flash N          Frames the visual saccade indicator persists after an event (default: 12).

Pupillometry plugin:
  --pupillometry        Enable pupillometry tracking.
  --pupil-mode {rgb,ir}
                        Measurement mode (default: rgb).
  --pupil-baseline N    Baseline frames for calibration (default: 90).
  --pupil-upscale F     Upscale factor for RGB mode (default: 2.0).
  --pupil-ir-thresh N   IR threshold for dark-pupil segmentation (default: 40).
  --pupil-filter {kalman,ema}
                        Ratio smoothing filter (default: kalman).
  --pupil-ema-alpha F   EMA smoothing alpha -- only used when --pupil-filter is ema (default:
                        0.3).
  --pupil-kalman-process-noise F
                        Kalman process noise -- controls how fast the filter adapts to ratio
                        changes. Only used when --pupil-filter is kalman (default: 0.0001).
  --pupil-kalman-meas-noise F
                        Kalman measurement noise -- higher values smooth more aggressively. Only
                        used when --pupil-filter is kalman (default: 0.01).
  --pupil-ear-thresh F  EAR threshold for blink detection (default: 0.21).
  --pupil-blink-frames N
                        Min consecutive low-EAR frames for blink (default: 2).
  --pupil-outlier-window N
                        Hampel outlier filter window size (default: 15).
  --pupil-per-eye       Enable per-eye (left/right) measurements.
